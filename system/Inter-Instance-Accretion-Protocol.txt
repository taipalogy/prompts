# Inter-Instance Accretion Protocol: The Master Pattern Library

This library is a core extension of the **Chain Accretion Framework**, fundamentally dependent on the **Shorthand Interface Protocol (SIP)**. It codifies the **IIA Protocol**, where sovereign **Modules** are deconstructed and recomposed via **Synthesized Interfaces**.

**Core Integration**: IIA is execution-agnostic. It governs **Structural Accretion** (logic/intent), while any **OpenClaw-like** technology provides **Physical Grounding** (commits/sync). IIA pours; the gateway grounds.

## 4. Operational Laws of the IIA Protocol

1. **The Law of Synthesized Interfaces**: Interfaces are "poured" at runtime based on **Type Synthesis** and deconstruction.
2. **The Law of Material Density**: Every `reduce` must increase type-safety and harden security.
3. **The Law of Primitive Selection**: All primitives (**Functional** and **Shell**) are **L0 Execution**. Use **Functional** for data-shaping and **Shell** for file-shaping. **L1+ Orchestration** must only **Dispatch** or **Curry** these tools, never shovel directly.
4. **The Law of Gateway Agnosticism**: Intent is independent of the commit tool (OpenClaw or similar).
5. **The Ascension Trigger**: Deploy **Cursor** for L0. **Ascend** to **Claude Code CLI** for **Logic Decoupling**, **Autonomous Audit**, or **Security Hardening**.
