# Logic Circuit: THE ADJECTIVE-EDITING (v1.0)
# Role: Surgical Identification & Mutation of Semantic Material
# Dependency: Noun-Lock_Protocol / Material_Stock_Integrity

[[ SCHEMA: Material_Stock + Adjective_Index = State_Collapse ]]

## 1. Identification Logic (The Diagnostic Grid)
- [Scan]: Locate all modifiers (Adjectives) within the Material Stock.
- [Index]: Assign a 2-digit coordinate (01-XX) to each modifier.
- [Anchor]: Map each Index to its specific Noun (Core Anchor).
- [Filter]: Decompose Noun-Phrases (e.g., "Dark Fantasy"). 
  - [Adj] (Dark) = Index(XX)
  - [Noun] (Fantasy) = Locked_Material_Stock

## 2. Declarative State Changes (The Edit Gates)
- [Subtract]: Index(XX) -> expand:Negative_Space (Remove)
- [Shift]: Index(XX) -> accrete:Attribute(New) (Replace)
- [Shared_Anchor]: Index(XX) + lock:Shared_Anchor -> Prevents mutation of the targeted Index during a ripple.

## 3. The Dependency Ripple (Autonomous Suggestor)
- IF Index(XX) is [SUBTRACTED] or [SHIFTED]:
    - SCAN adjacent Indices (XX-1, XX+1) for ripple impact.
    - IF adjacent Index has [lock:Shared_Anchor] -> BYPASS mutation.
    - ELSE -> Perform [VALENCY_SYNC] (Article/Conjunction correction).

## 4. Noun-Phrase Integrity Rule
- [Anchor_Retention]: In any [Adj + Noun] pair, the [Noun] is Hard-Locked as Material Stock.
- [Surgical_Mutation]: Only the [Adj] is mapped to an Index (01-XX).
- [Weld]: [Shared_Anchor] prevents the [Adj] from being altered by neighboring ripples, effectively preserving the specific "flavor" of the Noun without altering the Noun itself.

## 5. The Synthesis Formula (User Interface)
- [Formula]: [PORTRAIT_PROMPT_PASSAGE] + [ADJECTIVE_PROMPT_PASSAGE]
- [Example 1]: "...round or oval glasses..." + "Suggest Mutation_Manifold for Index(21)"
- [Example 2]: [Formula]: portrait.txt + adjective-editing.txt
- [Logic]: Triggers autonomous mapping and alternative attribute generation.
- [Constraint]: REJECT all Noun-Phrases as mutation inputs. DECOUPLE [Adj] from [Noun] if compound input is provided.
