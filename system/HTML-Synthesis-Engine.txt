# HTML Synthesis Engine - Preview

The **HTML Synthesis Engine** is a logic-driven framework for the modular assembly of ephemeral HTML structures.

**Primary Design References:**

* [Markdown is dead, long live HTML](https://x.com/trq212/status/2052809885763747935)

## 2. Composition Shorthand: Syntax Map

### Available Operations (Operators)

| Sigil | Operation | Functional Role |
| --- | --- | --- |
| `+` | **Accretion** | Connects Verbs and Nouns to the chain. |
| `:` | **Targeting** | Identifies the specific Noun or Verb to act upon. |
| `@` | **Styling** | Applies a Property or Material to the preceding element. |
| ` | ` | **Piping/Parallel** |
| `()` | **Grouping** | Defines logic blocks, specifically for `branch` operations. |
| `->` | **Materialization** | The final trigger to compile the polymer into an `.html` file. |

### Available Chain Members

* **Initializers**: `anchor:Document`, `anchor:Tool`
* **Structural**: `nest:`, `sibling:`, `wrap:`, `branch:`
* **Logic/Data**: `utilize:lib()`, `utilize:agent()`, `hydrate:state()`, `sync:state()`, `tether:logic()`, `tether:method()`
* **Aesthetic Presets**: `@ Vellum`, `@ Anodic`, `@ Serif`, `@ Tactile`

## 3. Operational Modes (Agnostic)

### Archetype A: Editorial_Layout (Static or Dimensional)

* **Trigger**: `anchor:Document`.
* **Layout**: Fragmented State (**64px**). Can be completely static or accretion interactive `portals`.

### Archetype B: Tool_Chassis (Utility or Sandbox)

* **Trigger**: `anchor:Tool`.
* **Layout**: Fused State (**0px**). Scales from simple calculators to multi-branch `hot_reload` sandboxes.

---

## 4. The Rules (The Physics)

**Item 1: The Ephemeral Protocol**
The final output is a single, portable HTML file. It requires no server or build-step to function. All logic is client-side.

**Item 2: The Kinetic Docking Law (0/64)**
Layout spacing is non-negotiable. Modules are either fused (**0px**) or fragmented (**64px**). This ensures the "Technical Luxury" aesthetic regardless of functional density.

**Item 3: Dimensionality Agnosticism**
Interactivity is an optional choice. The engine may be used to generate a perfectly knolled, static document that simply uses HTML as a "superior paper" for typography.

**Item 4: Zero-Secret Mandate**
Tools requiring API keys must use an input gate. Hardcoding credentials remains the only grounds for a Protocol Failure.

## 5. Implementation Archetypes (Examples)

> **Ex 1: The Static Specimen (Pure Paper)**
> `anchor:Document @ Vellum + nest:text(Essay) @ Serif -> materialize:Static_Specimen.html`

> **Ex 2: The Markdown Editor (Reactive Utility)**
> `anchor:Tool @ Anodic + utilize:lib(MarkedJS) + (branch:Editor_Pane | branch:Preview_Pane) + nest:interface(Textarea) + tether:method(hot_reload) + sync:state(LocalStorage) @ Vellum -> materialize:Markdown_Editor.html`

> **Ex 3: The Prompt Sandbox (Multi-Stage)**
> `anchor:Tool @ Anodic + nest:chassis(Multi_Stage) + (branch:Source | branch:Variables | branch:Output) + nest:annotation(Unicode_Symbols) + tether:method(hot_reload) + hydrate:state(URL_Hash) + sync:state(LocalStorage) @ Tactile -> materialize:Native_Sandbox.html`
